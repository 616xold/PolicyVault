# $docs-runbook-keeper

Use this skill when behavior changes affect the local demo, scripts, or interview explanation.

## Responsibilities

- keep `docs/ops/demo-runbook.md` current
- keep copy-paste commands accurate
- update architecture docs if the state machine or public API changed
- record surprises that would matter in a live demo
- make the README truthful about what is and is not implemented

## Output standard

A new reviewer should be able to clone the repo, run the demo flow, and understand the happy path without asking follow-up questions.
