# $demo-proof-auditor

Use this skill when shipping the demo script, README, screenshots, or interview runbook.

## What good looks like

- `demo.ts` prints state transitions in a readable order
- a fresh local chain can reproduce the flow
- the README explains what the project is and why the design is intentional
- the demo includes at least one revert path
- the runbook tells the operator exactly which buttons or scripts to run
