# $execplan-orchestrator

Use this skill whenever the change spans multiple files or a full submilestone.

## Responsibilities

- read the active ExecPlan before coding
- keep work constrained to the current submilestone
- update `Progress` after meaningful progress
- record any changed assumptions in `Decision Log`
- do not silently broaden scope

## Required report at each stop

1. files changed
2. commands run
3. results observed
4. remaining risks
5. exact next step
